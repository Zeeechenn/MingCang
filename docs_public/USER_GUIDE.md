# User Guide

这份指南回答一个问题：**怎样用自然语言和明仓完成研究任务。** 明仓是 agent，不是让用户背一组内部入口的脚本集合。除安装和 demo 启动外，日常使用都推荐直接对明仓说人话。

> 当前推荐：优先使用 `mingcang` Agent。前端在大规模重构后仍在体验优化中，可以作为本地可视化查看入口，但不是当前主推使用方式。

## 2. 最快上手

### 2.1 安装并进入 Agent

```bash
curl -fsSL https://raw.githubusercontent.com/Zeeechenn/MingCang/main/scripts/install.sh | sh
mingcang
```

进入后直接说：

```text
帮我看一下 300308，结合信号、新闻、长期标签和过去记忆，告诉我现在该观察什么风险。
```

明仓会自己读取本地上下文、调用需要的研究能力，并把结果翻成可读的研究结论。你不需要知道内部工具叫什么。

### 2.2 可选：只体验 demo

如果你还不想接真实数据，只想看一遍流程，可以在仓库目录里启动 demo：

```bash
make demo
```

然后打开：

```text
http://127.0.0.1:5173
```

demo 会准备示例自选、价格、信号、持仓、长期论题、复盘和待确认记忆候选。它的意义是展示研究闭环，不是真实投资建议，也不代表你的真实关注列表。

## 3. 日常对话工作流

### 3.1 研究一只股票

适合这样问：

- “帮我看一下 300308，现在是观察、试错还是规避？”
- “这只票最近最大的风险是什么？有没有新闻或记忆提示？”
- “官方信号和研究 copilot 的看法是否冲突？冲突在哪里？”

你会拿到：

- 最新官方信号和风险线。
- 最近新闻、情绪和数据新鲜度。
- 长期标签与研究 copilot 影子结论。
- 与这只股票相关的历史记忆、已验证风险和待验证问题。

判断边界：

- 官方信号是规则结果。
- copilot 是研究影子意见，不能覆盖官方信号。
- 记忆用于补上下文，可信度取决于是否经过结果验证和人工确认。

### 3.2 每天扫描关注列表

适合这样问：

| 场景 | 直接说 |
|---|---|
| 盘前 | “盘前扫一遍我的关注列表，告诉我今天先看哪些风险和事件。” |
| 盘中 | “帮我看一下盘中有没有风险线接近、新闻异动或需要记下的观察。” |
| 盘后 | “收盘后复盘一下今天的信号、新闻、风险线和待跟进事项。” |
| 周末 | “做一次周末体检，看看长期论题、错过的触发器和本周复盘。” |

明仓会尽量把报告里的内部术语翻成研究问题，例如“需要验证订单兑现”“风险线过近”“新闻情绪与价格背离”。这些输出是研究纪律，不是自动交易指令。

### 3.3 管理关注列表和候选池

适合这样问：

- “把中际旭创加进关注列表，后面每天帮我盯着。”
- “把最近没有数据的票列出来，告诉我哪些需要补资料。”
- “扫一遍关注列表，按今天最值得研究的顺序排一下。”
- “先预览要写入的关注列表变化，我确认后再执行。”

边界：

- 关注列表是本地研究池，不等于买入清单。
- 写入前可以先让明仓给你预览。
- 加入关注不会改变信号权重，也不会触发真实交易。

### 3.4 做专题或公司深度研究

适合这样问：

- “研究一下 AI 光模块景气链，重点看 800G / 1.6T 需求是否还能兑现。”
- “把先进封装作为长期主题跟踪，列出核心证据、反证和失效条件。”
- “这篇外部观点是否值得进入观察哨？帮我找反方证据。”
- “用多空辩论的方式看一下这只票，先说多头逻辑，再说最强反方。”
- “用 Serenity 的方式检查一下供应链瓶颈、证据等级和非共识线索。”

深度研究会整理本地行情、新闻、财务、已有研究和配置好的外部搜索/LLM provider。它不会因为一篇材料讲得好就直接改变官方信号。

### 3.5 让 LLM 做裁量参考

适合这样问：

- “把今天的候选票做个比较，哪些只是观察，哪些更适合小仓试错？”
- “这只持仓现在是继续观察、降仓倾向，还是应该准备离场？列证据和反证。”
- “给我一个加减仓时机参考，但不要替我下单，也不要改官方信号。”
- “把这次判断做成裁量参考卡，附上再评估触发条件。”

边界：

- LLM 裁量层默认灰度关闭；开启后也只是 reference / observe-only。
- 它可以做候选比较、持仓去留解释、时机提示和复盘提炼。
- 它不能修改官方信号、止损、止盈、仓位，也不能替你买卖。

### 3.6 建立长期论题

长期论题适合这些问题：

- “这个行业的慢变量是否变化？”
- “某个外部研究者的判断怎样持续跟踪？”
- “什么情况说明这个逻辑已经失效？”

你可以这样说：

```text
把“AI 光模块需求仍在上行”建成长期论题，帮我写清楚来源、失效条件、跟进指标和复盘节奏。
```

长期论题是研究态，不是买入信号。它负责提醒你持续验证，而不是替你下结论。

### 3.7 复盘和记忆

适合这样问：

- “复盘这次宁德时代亏损，根因是什么？能不能沉淀成一条规则？”
- “过去我在半导体票上最常犯哪些错误？”
- “这条经验现在是待确认还是可信记忆？为什么？”
- “把这次复盘候选先列出来，我确认后再写入记忆。”

记忆规则：

- LLM 输出默认只是候选。
- 复盘归因后才可能产生可升级记忆。
- 人工确认并经过结果验证后，才进入可信记忆。
- 可信记忆会在下次研究相同股票或主题时自动提醒你。

### 3.8 管理持仓和风险

适合这样问：

- “帮我看一下当前持仓里，哪些风险线离得最近？”
- “这只票的止损/止盈是怎么来的？和 AI 判断有没有关系？”
- “如果我记录一笔纸上持仓，会影响真实交易吗？”

边界：

- 持仓记录用于研究和复盘，不连接券商。
- 明仓不自动交易，不替你下单。
- 止损/止盈来自 ATR 和风险约束，不是 AI 价格预测。

## 4. 前端怎么用

前端可以作为本地可视化查看入口，但当前仍在重构后的体验优化阶段。建议先用 agent 完成任务，再用前端查看案卷、信号、复盘和来源健康。

| 页面 | 用途 | 优先看什么 |
|---|---|---|
| 脉冲 | 自选、搜索、候选池、最新状态 | 今天要研究哪只票。 |
| 单股详情 | 单票完整研究视图 | 官方信号、新闻、证据、copilot、记忆。 |
| 复盘 | 每日和长期复盘 | 结论是否兑现，是否产生记忆候选。 |
| 持仓 | 记录持仓和风险 | 市值、成本、浮盈亏、止损止盈。 |
| 聊天 | AI 项目助手 | 问项目、生成候选动作、确认执行。 |
| 配置 | 系统参数 | 权重、阈值、LLM、数据覆盖、kill switch。 |

## 5. 更好的提问方式

| 目标 | 推荐问法 |
|---|---|
| 要结论 | “先给结论，再列证据和反证。” |
| 要风险 | “不要只说机会，重点说失效条件和下跌风险。” |
| 要复盘 | “按当时信息、实际结果、根因、可沉淀规则来复盘。” |
| 要行动纪律 | “只给观察点和风险线，不要替我下单。” |
| 要写入本地状态 | “先预览变更，我确认后再写入。” |

## 6. 出问题时怎么问

- “帮我做一次健康检查，并用普通话解释哪里不正常。”
- “为什么这只票没有最新信号？是数据缺失、没在关注列表，还是任务没跑？”
- “这次结果能不能复现？请告诉我数据日期和来源。”
- “我不想改任何本地状态，只做只读诊断。”
