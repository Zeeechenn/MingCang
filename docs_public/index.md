# 明仓

本地优先的 A 股研究 Agent。把自选、行情、新闻、官方信号、AI 研究、长期论题、复盘和记忆，放进同一个**可对话、可审计**的研究流程。

> 当前推荐：优先使用 `mingcang` Agent，直接用自然语言和明仓对话。前端在大规模重构后仍在体验优化中，可以作为本地可视化查看入口，但不是当前主推使用方式。

## 快速开始

=== "安装 Agent"

    ```bash
    curl -fsSL https://raw.githubusercontent.com/Zeeechenn/MingCang/main/scripts/install.sh | sh
    mingcang
    ```

    进入后直接说：

    ```text
    帮我看一下 300308，结合信号、新闻、长期标签和过去记忆，告诉我现在该观察什么风险。
    ```

=== "体验 demo"

    ```bash
    make demo
    ```

    启动后打开浏览器：

    ```text
    http://127.0.0.1:5173
    ```

    demo 只用于体验研究闭环，不代表你的真实关注列表或研究结论。

## 你可以直接这样说

| 你想做什么 | 直接对明仓说 |
|---|---|
| 研究一只股票 | “帮我看一下 300308，现在适合观察、试错还是规避？” |
| 盘前检查 | “盘前扫一遍我的关注列表，告诉我今天先看哪些风险。” |
| 盘后复盘 | “收盘后复盘一下今天的信号、新闻和需要跟进的票。” |
| 维护关注列表 | “把中际旭创加进关注列表，后面每天帮我盯着。” |
| 跟踪一个主题 | “把光模块 1.6T 需求作为长期论题跟踪，列出失效条件。” |
| 复盘和记忆 | “复盘这次亏损，看看有没有能沉淀成规则的东西。” |

## 明仓是什么？

明仓**不是** AI 荐股器，也**不是**自动交易系统。它不接券商，不自动下单，不让 LLM 替你买卖。

它更像一个研究操作台：把每只股票的价格、新闻、官方信号、长期论题、AI 研究、复盘记录和记忆上下文放在一起，让你能看清一次判断来自哪里、哪些内容只是影子研究、哪些动作会写入本地状态。

## 核心功能

<div class="grid cards" markdown>

-   __Agent 对话入口__

    ---

    用自然语言完成研究、扫描、关注列表、复盘和记忆查询。

-   __单股研究__

    ---

    聚合官方信号、新闻、长期标签、research copilot 和记忆上下文。

-   __每日扫描__

    ---

    按盘前、盘中、盘后和周末节奏整理关注列表、信号、新闻、风险线和待复核事项。

-   __盘后风险面板__

    ---

    把候选、持仓体检、风险警示、保护动作和财务质量旗标放进一个复盘视图。

-   __LLM 研究__

    ---

    用于资料整理、多轮辩论、反方质询、风险提示和候选动作生成，不覆盖官方信号。

-   __LLM 裁量增强臂__

    ---

    围绕候选比较、持仓去留、加减仓时机和复盘提炼输出参考卡；默认灰度关闭，开启后也只做 observe-only。

-   __长期论题__

    ---

    记录外部判断、失效条件、跟踪指标和复盘节奏。

-   __复盘记忆__

    ---

    通过复盘归因产生候选，经人工确认和结果验证后升级可信记忆。

-   __数据系统__

    ---

    行情、新闻、财务、provider 健康和只读 global data，强调可追溯和可复核。

-   __Serenity 瓶颈框架__

    ---

    作为供应链瓶颈、证据等级、非共识线索和反方问题的研究检查清单，不接管生产信号。

</div>

## 当前体验状态

| 入口 | 建议 |
|---|---|
| Agent | 当前主推入口。适合日常研究、盘前盘后检查、复盘、记忆查询和关注列表维护。 |
| 前端 | 可查看案卷、信号、复盘和来源健康；重构后仍在体验优化中，暂不作为主推入口。 |
| 底层接口 | 给维护者、外层 agent 和开发者使用；普通用户优先看 [User Guide](USER_GUIDE.md)。 |

## 核心边界

| 边界 | 行为 |
|---|---|
| 官方信号 | 规则系统输出，主要由技术、情绪和风控组成。 |
| LLM 研究 | 负责整理、反问、辩论、候选比较、时机解释和风险提示；默认不覆盖官方信号。 |
| LLM 裁量增强臂 | 能辅助候选内排序、持仓去留和复盘提炼；默认灰度关闭，开启后仍只输出参考卡。 |
| 量化系统 | 当前是验证和影子证据路径，不进正式信号。 |
| 写入动作 | 自选、持仓、配置、记忆等高风险动作必须显式确认。 |
| 交易执行 | 不接券商，不自动下单。 |
