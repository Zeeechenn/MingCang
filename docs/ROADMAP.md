## M65 研究可信度收口（2026-07-15）

M65 优化的是研究报告的**证据可见性、审计性与可复现性**，不优化交易信号，不承诺 alpha。
FinceptTerminal 等外部项目只提供组织形式参考；明仓继续复用现有 report-pack、ResearchReportGate、
M57 记忆和 M63 编排，不新建第二套 DataHub、缓存、记忆库或人格平台。

**优化前基线（Phase 0 冻结口径）**：

- 当前库共有 2,355 条 `decision_runs`，其中 29 条 deep-research；15 条保存了
  `gate_status=pass`，14 条历史记录没有 gate 状态，未发现 gate warning。
- `stock_memory_items` 1,638 条、`memory_recall_index` 1,353 条、`memory_atoms` 3 条；
  Serenity 结构化记录、warning 与 fingerprint 均为 0。
- Serenity 旧独立分析器从未进入 CLI / Web / scheduler / pipeline 的生产调用链，当前
  `analyze()` 已退役并返回空；因此不能声称其历史上产生了可量化收益。现有价值主要来自其
  5/6 方法步骤已并入证据、时序、反证、边界和报告门，独立贡献尚未被实验识别。
- Evidence Ledger 当前会用 `fetched_at` 回填 `published_at`，抓取时间可能被误读为发布时间；
  Gate 结论主要是自然语言 reasons / warnings，机器难以稳定比较具体失败项。

### Phase 0：基线与审计定义

- 固定同一 `as_of`、同一输入证据与同一输出模板；为后续四臂评估准备离线可复现样本。
- Gate 检查项使用稳定 `check_id`、状态、严重级别与证据引用，同时保留旧的
  `status / reasons / warnings`，避免破坏现有调用方。
- 每次评估保存臂名、配置版本、输入指纹、耗时/调用预算和逐项指标；缺失数据必须显式，
  不允许静默补成“已验证”。

### Phase 1：Evidence Card + 结构化报告门

- Evidence Card 明确区分 `published_at` 与 `fetched_at`，显示 `as_of` 下的新鲜度、来源层级、
  可用性、风险标记和缺失原因；没有发布时间就显示缺失，不再用抓取时间伪装。
- ResearchReportGate 输出机器可读检查明细，至少覆盖来源、时序、关键数据、叙事、措辞和
  中文风格；前端/审计工具可直接定位失败门，而不是解析自然语言。
- Phase 1 验收：旧接口兼容、聚焦测试通过、Evidence Card 时间语义正确、结构化结果与总 verdict
  一致，且信号边界 diff=0。

### M57 前置门：真 trace 与治理台实操

Phase 2 之前必须用真实 research trace 完成 miner 首跑，并在治理台完成一次查看、保留/拒绝或
晋升操作演练。所有候选默认 `pending`，自动 `trusted` 必须为 0；任何自动晋升都会阻断 M65。

### Phase 2：Base / Memory / Serenity / Both 四臂影子评估

- 四臂共享同一证据快照与 `as_of`：Base、Base+Memory、Base+Serenity 方法镜头、
  Base+Memory+Serenity。Serenity 不调用已退役分析器，不参与评分/投票，只输出结构化反证、
  边界与未知项。
- 最少 20 个案例、覆盖至少 3 个行业；先用确定性离线夹具验证评估器，再对真实研究样本影子运行。
- 主要指标：来源忠实率、关键事实覆盖率、矛盾处理率、可证伪性、幻觉/事实错误率；同时记录
  延迟与调用成本。只比较预注册指标，不采纳单个“看起来更好”的案例。
- **通过门**：相对 Base，Both 在关键质量指标上提升至少 15 个百分点，幻觉/事实错误率不恶化；
  默认日常额外 LLM 调用为 0，显式 deep 模式成本增幅不超过 25%，且信号边界 diff=0。
- **失败裁决**：任一硬门失败即停止 Phase 3-4，保留 Phase 0-1 的通用可信度改进，Serenity
  保持文档/人工方法镜头，并向 owner 汇报失败指标和样本限制。

#### Serenity skill 后续优化顺序

1. **本轮冻结与裁决已完成**：当前 skill / prompt digest、证据快照、`gpt-5.5/low`、输出模板和

4. **证据绑定与不确定性**：每个关键论断独立绑定来源等级、独立性与 freshness；转载去重；无法
   证实时明确输出 `unknown`。只允许定性概率方向，不生成 0-100 主观分、后验数值或聚合大分。
5. **修正评审参考集**：本轮评审只把原始证据包作为 ground truth，导致 Memory/Both 引用虽有

   “可核验记忆附录”，但仍隐藏哪一方案实际使用了它；否则不能公平量化 Memory 或交互效应。
6. **重新预注册可达门槛**：本轮 Base 已在 86.0%–90.85%，绝对提升 15pp 对部分指标数学上不可达。

   rubric，并增加第二名独立盲评审。仍保留幻觉不恶化、成本 ≤25%、交易边界 diff=0 等硬门。
7. **仅在新预注册后同池复测**：使用相同 20 例，同时单列真正适用 Serenity 的不少于 3 行业子集。
   Serenity 专项目标仍应是矛盾处理与可证伪性稳定正向、来源忠实率/事实覆盖不下降、幻觉不恶化；
   未达到前继续文档/人工镜头，不进入 Phase 3-4。

## M64 Live Track 安全里程碑（2026-07-14）

M64 只交付本地决策支持前的安全漏斗，不接券商、不下单，也不修改信号、价格、持仓等
数据库状态。公开仓库仅保留 `live_trading/live_funnel.py` 与脱敏 state schema；实际
universe/state、`LIVE.md` 钱仓台账、运行日志和生成 subset 均由 `.gitignore` 默认隔离。

**代码完成条件（已满足）**：

- 持仓唯一来源为指定的本地 JSON state，不在代码中硬编码；state 日期必须等于交易日，
  且 state 中的持仓必须属于 live universe。
- SQLite 强制以 `mode=ro` 打开；脚本只允许写调用者显式传入且不与任何输入同路径的
  `--output` 子集路径。
- live universe 与 test2 universe 任一代码重叠即 fail-closed。
- 任一持仓缺指定交易日 price bar 或 signal 即 fail-closed；非持仓数据不完整则排除并
  输出 warning，不静默进入子集。
- state 记载的总持仓市值 / 组合总值超过 80% 即 fail-closed，80% 为不可通过 CLI
  放宽的代码常量；零组合总值或负持仓市值视为非法输入。
