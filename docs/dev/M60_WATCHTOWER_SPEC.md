# M60 观察哨 — 检测-响应型选股流水线 Spec v0(2026-07-03)

> owner 定调:选股入场公式证伪后,选股不能退回纯人工——改为**研究定方向 + 实时掌握
> 市场信号 + 第二时间跟进**。预测能做多少做多少,但检测必须做满。
> 与被证伪路线的本质区别:不再对全池日频排序找 alpha("谁会涨"),只对**研究圈定的
> 观察清单**做事件检测("它开始动了没有")——方向先验来自基本面研究,时机锚定在事件。

## 流水线(四层)

1. **方向层(慢,周频)**:深研 / 赛道研究员(track-analyst) / serenity / 景气 / 长期分析师团
   → 产出**观察清单**:赛道 + 个股 + 看多论点 + 验证条件 + 失效条件。
   复用 ATLAS 留下的 `forward_thesis` / thesis ledger 结构(基础设施变现,软联动升级版)。
   案例:owner 跑创新药板块研究 → 药明康德数日后启动 → 属"清单内启动",本应触发。
2. **观察哨(快,盘后确定性,零 LLM)**:只扫观察清单内标的——
   价量异动(涨幅分位/量比/N日新高突破)、**板块共振**(同主题多股同日强,单股异动可能是噪声、
   板块共振是信号)、新闻 L1 触发(M54 已建的 `news_trigger.decide_trigger`,直接复用)。
3. **确认层(第二时间,LLM 裁量)**:触发后带全上下文裁量——研究论点还成立吗、异动归因四路
   (公司事件/政策/行业/情绪,M54 归因卡结构)、估值/拥挤度提示 → 面板"跟进候选"卡,
   附裁量理由与风险,owner 拍板。
4. **入场纪律(确定性)**:第二时间规则——触发次日不追高开、回踩/放量确认等 2-3 个变体
   (参数由 M58 出场通道同款方法回测定);入场即挂 M58 出场参数。

## 诚实边界(必须先说破)

- "第二时间跟进"本质是条件化趋势跟随。刚证伪的是**全市场横截面动量排序**;本方案是
  **研究先验 + 事件锚定的小清单跟随**——条件概率不同,但**仍未被验证**,不得因直觉合理就免检。
- 验证方式:预注册假设"观察清单内触发标的的第二时间入场,5/10/20 日收益显著优于
  ①清单外同日触发标的 ②清单内未触发标的"。台账从面板每日记录自动攒(前向),
  并回溯 3-5 个历史案例(创新药→药明康德为第一个)做案例研究(exploratory,不作证据)。
- 停止条件:不自动下单;触发≠买入指令;清单必须带失效条件;LLM 裁量不可推翻出场纪律。

## 实施切片

- Phase 0:观察清单 schema(复用 forward_thesis 字段)+ 手动把现有研究产出(创新药等)录成首批清单。
- Phase 1:盘后观察哨 CLI(价量异动+板块共振+新闻触发,零 LLM)→ 面板加"跟进候选"区。
- Phase 2:LLM 确认层接进面板;第二时间入场规则回测(M58 通道复用)。
- Phase 3:前向台账攒 4-8 周 → 按预注册假设裁决。
