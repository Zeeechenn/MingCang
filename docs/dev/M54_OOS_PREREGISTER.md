# M54 新闻层 v2 — 干净 OOS 预注册判据（先于跑数落盘）

> 2026-06-28 落盘。承接 M52 收口（标题级情感无 IC）与 M54 设计 spec。**先写阈值与窗口，结果出来只对照、不事后挑解释**（守干净 OOS 纪律，防 p-hacking）。
> 全程 observe-only / 生产 diff=0 / 独立 OOS cache 命名空间 `oos_news_v2` / 不动 live test2 / 不污染 M52 的 legacy+capable 候选判据。

## 0. 前置数据依赖（必须先满足才有意义）
v2 的价值在"读正文"。但 DB 现有历史 news 行 **content 全为 NULL**（content 列在阶段0 才加）。东财正文回填**仅 ~59 天**（探针实测到 2026-04-29）。
**因此跑前必须先做内容回填**：对 universe 在窗口内回填东财正文（覆盖 ~Apr29–今），否则 v2 会因 content_status=title_only 大量走降级（DEGRADED/NEWS_THIN）→ 不是对"正文级"的真实检验。
- 若回填覆盖足 → 用历史窗口 OOS。
- 若覆盖不足 → 改"从现在起向前采集正文"，数周后样本够再 OOS。
**回填覆盖率（有正文窗口/总新闻窗口）是跑前的 go/no-go 门：< 50% 不开跑，改向前采集。**

## 1. 固定设置（跑前锁定）
- **Universe**：`paper_trading/test3_universe_50.json`（50 支）。
- **窗口**：跑前据回填覆盖确定；优先 2026-04-29..（今-7d）以匹配东财回填可达范围。
- **Lookback**：3 天。**Horizons**：h3d / h5d。
- **缓存隔离**：ns `oos_news_v2`，禁止写生产 sentiment_cache。
- **DEGRADED 排除**：degradation_flags 含 DEGRADED 的窗不进 IC 主样本（harness 已实现 skip + 计数）。

## 2. 对照腿（同窗口/同 universe/同缓存隔离，各自独立 ns）
| 腿 | 含义 |
|---|---|
| legacy-fast | M52 收口基线（全标题、便宜档） |
| legacy-capable | legacy 广度 + capable 模型（M52 已证伪此杠杆为负，作下界对照） |
| **news_layer_v2** | 本里程碑候选：正文级 + 分级抽取 + source_diversity 加权 + 资金流融合 |

## 3. 主目标：相对排序
同一样本下 news_layer_v2 vs legacy 的 IC/ICIR 排序。**主裁决看相对排序**（M52 教训：绝对值易虚高，稳健的是排序）。

## 4. 绝对晋级门（次要，过不了也不影响主裁决，但上 live 必须全过）
- IC mean ≥ 0.04 / ICIR ≥ 0.40 / 分桶单调 / 非重叠 stride IC 天数 ≥ 20 / 足量新鲜样本 / fallback<10% / DEGRADED 已排除。

## 5. 预注册决策规则（跑前写定）
1. **news_layer_v2 显著 > legacy（Δ IC ≥ +0.02 两 horizon）且过绝对门** → v2 列为候选（仍需用户显式 epoch-reset 授权 + test2 边界才上 live）。
2. **v2 ≈ legacy（Δ<0.02）或仍为负** → v2 正文级也无 alpha；**复证 M52 根本结论"新闻短周期无预测力"**；保留 v2 代码与 harness 作资产，生产维持 legacy，转二期（公告/龙虎榜事件路线）或将新闻降级为上下文。
3. **v2 优于 legacy 但不过绝对门** → 有方向但样本/质量不足，扩样本/扩源（iFinD/公告）后再预注册一轮。
4. 任一上 live 前置：绝对门全过 **且** 用户显式授权。本轮只出研究裁决。

## 6. 诚实 caveat（跑前写定）
- 东财正文 ~140 字（是正文 lead/摘要，非超长全文）；回填仅 ~59d；50 支样本 + 短窗，h5d 非重叠 IC 天可能 < 20（样本硬限）——这会卡绝对门 h5d，但不否定主目标（相对排序）。
- v2 OOS 用 LLM 打分（capable 档），跨多个额度窗口、需数天；缓存接力、失败分不落（沿用 M52 纪律）。
- 绝对值不可信，验的是相对排序在干净 OOS 上能否站住。

|---|---|---|---|---|---|
| h3d | -0.013 | -0.06 | 12 | 否 | ✗ |
| h5d | +0.055 | 0.25 | 8 | 否 | ✗ |

**裁决（对照 §5）= 规则3：有方向但样本不足、不可信、不晋级。**
1. **gate_blocked**：两 horizon IC天 12/8 « 20 门、非单调、ICIR<0.40。
2. **h3d≈0；h5d +0.055 看似翻正**（M52 legacy-sonnet h5d 为 -0.061），但 **8 个 IC天的 +0.055 统计上不可信**——正是预注册 §6 警告的小样本虚高（参照 06-18 +0.32 假象）。**不能据此说 v2 赢了。**
3. **根因 = IC天太少**：东财独力内容（窗口正文覆盖 45.7%）+ DEGRADED 排除 696 窗 → 横截面过薄，与 §0/§6 预测一致。
4. **主目标（相对排序）未完成**：本轮只跑了 news_layer_v2，**未跑 legacy-fast/legacy-capable 同窗对照腿**（m54 harness 仅打 v2 分）。"v2 vs legacy"干净排序待补。

**下一步（要真定论需）**：① 提高内容覆盖——加 iFinD 正文抽取 / 向前采集（**不靠 Anspire**，符合摆脱 Anspire 目标）把 IC天推过 20；② 给 m54 harness 加 legacy 对照腿，同窗同样本比 v2 vs legacy；③ 再预注册跑一轮。当前证据**不足以启用 v2**，生产维持 legacy。

**下一步（拿 promotion 级定论）**：① **扩 universe 到 ~100-300 支**（更多横截面名 → IC天过 20，这是唯一硬门）；② 同样本重跑三腿 + 检查单调；③ 过门则按 §5 规则1 走用户授权 + epoch-reset 才上 live。相对排序已强烈提示正文层有真 alpha，扩样本验证优先级高。

## 9. 第三轮预注册（2026-07-02 落盘，先于跑数；跑后只对照不改判据）

**固定设置（跑前锁定）**：
- **Universe**：`paper_trading/test3_universe.json` 全 **100 支**（test3 家族上层集，M27.2 规则构建：min_bars 500 / 流动性 / 每行业 ≤12。沿用同一家族避免选择偏差；前两轮 50 支即其子集）。**不再另建更大 universe**——若本轮 IC天仍不足走规则2（扩窗口），不无限扩截面。
- **窗口**：2026-04-29 .. 2026-06-25（今-7d 保 h5d 实现；较第二轮多 ~3 个交易日）。
- **跑前修订（2026-07-02，打分启动前）**：三腿统一模型改为 **claude-sonnet-4-6（claude CLI capable tier，即 `local_cli_model_capable` 默认值；`LOCAL_CLI_PREFER_CODEX=false` + `LOCAL_CLI_NO_CODEX_FALLBACK=1` 保腿纯净，不回退 codex）**。影响：同轮三腿仍同模型（主裁决=轮内相对排序，干净不变）；**跨轮对比（本轮 v2-full vs 第二轮 codex/medium）叠加了模型变量，只作方向参考、不作定量对比**——与前两轮的 codex 基线不可直接数值比较，这是本轮的诚实约束。
- **三腿（同窗同 universe 同模型 claude-sonnet-4-6，ns 对齐）**：
  1. **v2-full**（与第二轮同构配置，pyramid off）——**主腿**，与第二轮直接可比；
  2. **legacy-fast**（标题级）——对照腿；
  3. **v2-pyramid**（`news_v2_pyramid_enabled=True`，阶段7 管线）——**次要腿**，回答两问：(a) 信号保真度——触发门/域共享/预算护栏对 IC 的损伤应有限（判据见规则4）；(b) 成本——实际 LLM 打分调用数 vs v2-full 的节省比（首个真实数据点）。
- **go/no-go**：回填后窗口正文覆盖 **≥50%** 才开跑；<50% 改向前采集数周。`provider=unknown` 的存量旧行（零正文旧摄入）不计入 v2 覆盖资产、按现有 DEGRADED 语义处理。
- **go/no-go 口径判定（2026-07-02 回填后、打分前落盘）**：回填实测——东财 100%（1032/1032）、iFinD 87.9%（501/570）、`unknown` 存量 2181 行 0 正文；总口径 40.5%。**关键事实：unknown 行为 M54 前 Anspire 摄入，阶段0 探针已证 Anspire 无历史正文可回填——对历史窗口，50% 总口径物理不可达，且与回填努力无关（本次回填仅 +1 行，因可回填源已近满）**。按本节预写条款以「内容可回填源口径」判 go：eastmoney+iFinD 合计 **95.7%（1533/1602）≥50% → GO**。总口径 40.5% 记为诚实 caveat：evidence 集中 58% 为 title-only 旧行，靠 DEGRADED 排除与 title_only 0.65 折扣兜住（与第一/二轮同构，第一轮总口径 45.7% 亦低于 50%）。唯一能抬总口径的路径是向前采集（新摄入带正文），不阻塞本轮。
- DEGRADED 排除 / IC天最小截面规则 / 失败分不落 / 缓存接力：全部沿用 §1-§2。

**主裁决与决策规则（跑前写定）**：
- 主裁决仍是**相对排序**：v2-full vs legacy，Δ IC ≥ +0.02 两 horizon 保持才算方向续存。
- 绝对门同 §4（IC ≥0.04 / ICIR ≥0.40 / 单调 / 非重叠 IC天 ≥20 / fallback<10%）。
1. **v2-full 过绝对门 + 显著>legacy** → 候选晋级（仍需用户显式授权 + epoch-reset 才上 live）。
2. **方向保持但 IC天 仍 <20** → 样本硬限未解，改**向前采集数周扩窗口**再预注册一轮；不再扩截面。
3. **方向消失**（任一 horizon Δ<+0.02）→ 第二轮"方向性胜出"判定为小样本假象，v2 信号设计回炉；生产维持 legacy。
4. **pyramid 保真度**：v2-pyramid 对 v2-full 的 IC 差 **< -0.01（两 horizon 均劣）** → 判触发阈值过紧回炉（只影响阶段7 调参，不影响 v2-full 主裁决）；差 ≥ -0.01 → 金字塔"省成本不伤信号"成立，第四轮起 pyramid 可为默认跑法。
5. 任何腿上 live 的前置不变：绝对门全过 **且** 用户显式授权。本轮只出研究裁决。

**诚实 caveat（跑前写定）**：新增 50 支的东财正文受 ~59d 回溯限制，窗口早段（4/29–5/4 附近）可能偏薄——由 DEGRADED 排除兜住，不人为剔窗；100 支 × 3 腿的打分为多额度窗口多天任务，缓存接力，中途失败分不落盘。

## 10. 第三轮结果（2026-07-02，claude-sonnet-4-6，100 支，窗口 04-29..06-25，三腿同 ns 对齐）

| 腿 | h3d IC/ICIR | h5d IC/ICIR | IC天(3/5) | 单调 |
|---|---|---|---|---|
| **v2-full** | +0.0544 / 0.49 | +0.0593 / 0.28 | 14 / 9 | h3d✓ h5d✗ |
| **legacy-fast** | **+0.1180** / 0.54 | -0.0222 / -0.08 | 14 / 9 | 否 / 否 |
| **v2-pyramid** | -0.0149 / -0.09 | -0.0995 / -0.48 | 14 / 9 | 否 / 否 |

**裁决（对照 §9 决策规则，不事后改判据）= 规则3：方向消失，v2 信号设计回炉，生产维持 legacy。**

1. **主裁决（相对排序）不再支持 v2**：h3d Δ(v2-legacy) = 0.0544-0.1180 = **-0.0636**（< +0.02 门，且为负——legacy 反而领先 v2）；h5d Δ = +0.0815（仍达标）。**两 horizon 不一致，且更看重的 h3d 反向**——按规则3 字面判定："任一 horizon Δ<+0.02"即触发，第二轮"v2≫legacy"的判定判为**小样本假象**，不再作为方向性证据。
2. **绝对门仍未过**（三腿一致）：IC 天 14/9 « 20，非单调。与第一轮(12/8)、第二轮(12/8)一样卡在同一处。
3. **横截面扩容再次验证无效**：universe 50→100 支，IC 天仅 12/8→14/9（+2 各），三轮数据一致证明**IC 天由非重叠时间窗口数决定，与横截面宽度基本无关**——这条此前只是推测，现在有三轮重复观测支持，可视为确定性结论：**下一步必须扩窗口（向前采集），不是再扩股票数**。
4. **pyramid 保真度未达标**：v2-pyramid 对 v2-full 两 horizon 均劣于 -0.01 门槛（h3d -0.069、h5d -0.159），触发规则4"触发阈值过紧回炉"。**但差距幅度远超预期**（不只是"损失一点信号"，是两个 horizon 都翻负）——不排除除阈值调参外还有实现层问题（例如未触发个股的"复用昨日结果"逻辑是否引入了系统性滞后偏差），需要专项复核，不能只当参数调优处理。此项**不影响本轮主裁决**（主裁决已由规则3 落定）。
5. **模型变量caveat**：本轮 v2-full/legacy 用 claude-sonnet-4-6，第二轮用 codex/medium——跨轮数值不可比，但**本轮内部 v2 vs legacy 是同模型同窗口严格对照**，规则3 的触发不受模型变更影响。

**结论**：v2 的"读正文≫读标题"假设**没能在样本扩大后复现**；h3d 上 legacy 反而更强。生产继续维持 legacy，不启用 v2。**下一步不是再扩样本或再调参**，而是：① 按规则2 改向前采集扩窗口（IC 天是唯一瓶颈，且与横截面无关已证三次）；② 但更优先的是先回答"为什么第二轮的胜出没能复现"——需要看 v2 打分的具体差异（是否第二轮 50 支恰好是有利子样本、还是本轮加入的 50 支拖累了信号）；③ pyramid 触发逻辑的过度损伤需要独立诊断，不与主裁决混同处理。当前 v2 未挣得启用资格，比第二轮更谨慎——不再是"方向确立、样本不足"，而是"方向本身存疑"。

## 11. 第三轮取证复核（2026-07-02 当晚，对 §10 裁决的修订 · 只读已缓存分数，零新 LLM）

§10 裁决落定后做了子样本取证（脚本 `scratchpad/analysis_abc.py`，复用 harness `_metrics_for_records`/`_forward_return`/`_load_prices`，全100 h3d 复现 -0.0636 与 §10 精确对上 → 方法学一致；leader 独立重跑 + 价格时间戳核验通过）。**取证推翻了 §10 的部分措辞，须修订。三点发现**：

**① h5d 跨腿对比本轮无效（数据完整性 bug，非信号问题）**：`prices` 表 fetched_at 证据——06-30/07-01/07-02 收盘价 `max(fetched_at)=07-02 19:13`，晚于 v2-full(14:44)、legacy(15:40) 跑完之时，且仅 24-25 支覆盖。窗口末段(≈06-18 之后)信号的 h5d 前向收益，三腿看到的是**不同的、跑时仍在补齐的价格快照**（pyramid 18:26-19:20 跑，用上了更全的价格，故其 h5d 能精确复现；v2-full/legacy 的 h5d 已不可逆向还原，复现值 v2 +0.029 / legacy -0.002 ≠ §10 报告的 +0.059 / -0.022）。**h3d 不伸到那么远、三腿精确匹配、不受影响 → 本轮唯一干净的跨腿对比是 h3d。§10 里凡用 h5d 的判断（含"h5d Δ=+0.0815 仍达标"）作废。**

**② 主裁决须按子样本修订：v2 在原50支上并未落败，反转完全由新增50支驱动（仅 h3d）**：

| h3d IC | 原50（=前两轮 universe） | 新增50 | 全100 |
|---|---|---|---|
| v2-full | +0.0203 | +0.1038 | +0.0544 |
| legacy | +0.0082 | +0.1955 | +0.1180 |
