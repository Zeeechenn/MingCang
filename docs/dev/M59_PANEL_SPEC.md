## 结构(四区 + 页头)

**页头**:as-of 日期 / 数据新鲜度(价格·新闻·标签各自最后更新)/ 当日降级 flag(如 LLM 预算超限、模型缺失)/ 市场 regime 一行(沪深300 趋势态)。

**① 买入候选区(LLM 裁量,治疗臂价值点1)**
- 输入:过阈值候选池(现行规则;将来=M58 选股位公式)。
- LLM 对候选做裁量排序,每只输出:一句论点、两条风险、验证问题、建议关注等级(watch/考虑买入)。**不给目标价、不给"强买"**。
- 附:该股 L0 记忆召回摘要(历史教训/论点,M57 通路)。

**② 持仓体检区(治疗臂价值点2+3)**
- 每只持仓:现价 vs 止损/止盈线、ATR trailing 当前档位、浮盈保护状态、距离触发的空间。
- LLM 裁量层:对"接近止损/高浮盈/论点变化"的持仓给决断建议(继续持有/减仓/清仓候选)+理由——test2 治疗臂清兆易式的主动决断,产品化在这。
- 加减仓时机:仅当候选池与持仓重叠或仓位限制允许时给出,附仓位约束核算(15%/30%/80%)。

**③ 避雷警示区(M58 避雷位)**
- 动量末档(现行=placeholder 公式 bottom20%)+ 持仓/自选与警示名单的交集高亮。
- 明确标注:预警≠卖出指令(注册假设 m58_stop_loss_momentum_tail_v1 的 forbidden_interpretation)。

**④ 复盘归因区(治疗臂价值点4)**
- 当日已实现交易:逐笔归因(入场依据是否兑现/出场类型/滑价成本)。
- LLM 提炼一条可复用纪律 → 写入 M57 记忆候选(pending,不自动 trusted)。
- 周频追加:本周账本 vs 同池等权基线、最大单票贡献占比(集中度警报)。

## 实现

- Schema:`postmarket_panel.v1` JSON(四区+页头字段)+ Markdown 渲染;落 `backend/tools/m59_panel.py` CLI(盘后手动跑,不接 scheduler,直至 owner 要求常驻)。
- LLM 调用走既有 provider 链与 llm_usage 预算桶;LLM 不可用时面板降级为纯规则版+显式 flag。
- 每区数据源解耦(模块化):选股/避雷/出场规则各自独立函数,M58 裁决后逐个替换不牵连。
- 测试:schema 校验、无 LLM 降级路径、仓位约束核算、L0 召回接线。
