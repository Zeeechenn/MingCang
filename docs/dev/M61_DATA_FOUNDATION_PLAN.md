## 2. 终态与原则

1. **服务层定位**:数据层服务最终判断;验收看判断质量(误判/漏判/错判减少),不看接入数。
2. **三车道分配**(每个能力必须落一道或明确拒绝):
   - ①回测因子道——PIT 严格(公告事件日/技术形态/资金流历史/龙虎榜)
   - ②面板/避雷道——当下准确即可(解禁/定增/监管事件、风险指标、股东变化)
   - ③研究裁量道——reference-only 绝不进打分(一致预期/盈利预测、板块联动、海外领先指标、F10)
3. **终态矩阵**:数据类别 × 源 × 消费模块 总表,每格 ∈ {已接入(道X)/测过不接(原因)/拒绝(原因)},**无"没看过"状态**。
4. **主源+fallback 保留**;每类数据 1 主 1-2 备,由 P1 体检结果定角色。
5. **显式降级铁律**:任何数据缺失/获取失败必须产生可观测事件(落库+面板告警),`except: return None` 全数歼灭。
6. **手册先行**:接入任何源之前,该源手册必须已入库;后续任何数据任务先查手册(写入 AGENTS.md 路由)。
7. **PIT 红线**:进①道的数据必须可按历史日取当时值;一致预期类明确禁入回测。
