# M60 Phase 2b — 第二时间入场规则影子台账(observe-only)规格

> 2026-07-05 立规格。动机:盲裁"长飞2月案"实证两套系统对高位强势股都只会"等回调"而共同踏空
> +105%(系统级盲区);M60 spec §入场纪律 已画出方向(触发次日不追高开、回踩/放量确认变体)。
> 红线:**observe-only 影子台账,零生产影响**——不改官方信号/持仓/面板买入建议,只记假想成交攒证据。

## 机制

新工具 `backend/tools/m60_second_entry.py`(CLI,盘后跑,只读行情+观察哨输出,写自己的影子台账 JSON)。

1. **输入**:M60 观察哨当日触发记录(paper_trading/m60_out/ 最新 watchtower 输出;字段自查)+ prices 表。
2. **对每个新触发标的,同时登记 3 个确定性入场变体**(全部次日起算,T=触发日):
   - V1 immediate:T+1 开盘价入场(基线变体,"第一时间")。
   - V2 pullback:T+1 起 5 个交易日内,首次日内低点触及 MA5(5日均线)当日,以 MA5 值入场;5 日内未触及则记 no_fill。
   - V3 confirm:T+1 起 5 个交易日内,首次"收盘创 T 日之后新高且成交量 > T 日量×0.8"当日,以该日收盘入场;未出现则 no_fill。
3. **出场口径**(统一,便于横比):入场后跟踪 5/10/20 交易日收益;另记 1.5×ATR14 初始止损是否被触发(触发则该变体以止损价终止,后续窗口收益按止损价冻结)。
4. **台账**:paper_trading/m60_out/second_entry_ledger.json——幂等追加(键=symbol+trigger_date+variant),每日盘后跑一次自动:登记新触发 + 回填已登记项的到期窗口收益。禁用 Date/随机以外没有约束,时间取 as_of 参数(默认库内最新交易日)。
5. **基线**:同日观察哨触发但"不入场"(收益恒 0)+ 等权持有全触发池 T+1 开盘入场(=V1 池化)——报告必须并列展示,防止把 beta 读成规则 alpha(仓库纪律:等权基线强制)。

## 预注册(实现完成后由执行者调用 m29 registry 工具登记,条目内容如下)

- hypothesis id: `m60_2b_second_entry_rules`
- 假设:观察清单内触发标的的 V2/V3 第二时间入场,20 日收益/回撤比显著优于 V1 立即入场(门:样本≥20 个触发、V2 或 V3 相对 V1 的 20日收益中位数差 > 2pp 且最大回撤不劣)。
- 状态:数据积累期,禁止在样本 <20 时下结论;窗口=前向,不回填历史触发(避免选择偏差——历史触发记录不含当时的确认卡语境)。
